## 6. 页面模块与所需接口

## 6.1 登录与身份模块

### 1) 获取当前用户

**接口**

`GET /api/auth/me`

**返回常见字段**

```json
{
  "id": "uuid",
  "username": "alice",
  "displayName": "Alice",
  "avatarDataUrl": "data:image/png;base64,...",
  "role": "USER",
  "status": "ACTIVE"
}
```

**前端用途**

- 判断是否已登录
- 渲染头像、昵称、角色
- `ADMIN` 用户跳转 `/admin`
- 控制“首页 / 管理后台”按钮文案

---

### 2) 登录

**接口**

`POST /api/auth/login`

**请求体**

```json
{
  "username": "alice",
  "password": "user123"
}
```

**返回**

返回内容与 `GET /api/auth/me` 基本一致，同时建立 Session Cookie。

**前端用途**

- 登录弹窗提交
- 登录成功后重新初始化页面

**常见错误**

- `400`：账号或密码为空
- `401`：账号或密码错误
- `403`：账号未启用或待审批

---

### 3) 更新个人资料

**接口**

`PATCH /api/auth/profile`

**请求体**

```json
{
  "displayName": "Alice",
  "avatarDataUrl": "data:image/png;base64,..."
}
```

**说明**

- `displayName` 可为空字符串
- `avatarDataUrl` 可为空字符串，表示移除头像
- 支持 `png / jpg / jpeg / webp / gif` Data URL

**前端用途**

- 个人资料弹窗保存
- 更新顶部头像和消息头像

---

### 4) 退出登录

**接口**

`POST /api/auth/logout`

**返回**

```json
{ "ok": true }
```

**前端用途**

- 清理登录态
- 跳回首页

## 6.3 聊天模块

### 1) 同步聊天

**接口**

`POST /api/chat`

**请求体**

```json
{
  "sessionId": "ses_xxx",
  "message": "什么是 BM25？",
  "topK": 6,
  "kbIds": ["kb_xxx", "kb_yyy"],
  "model": "model_uuid",
  "appendUser": true,
  "contextSize": 6
}
```

### 字段说明

- `sessionId`：为空时由服务端创建
- `message`：用户输入
- `topK`：总检索上限，当前前端固定传 `6`
- `kbIds`
  - `null` / 省略：表示使用当前用户全部可访问知识库
  - `[]`：表示关闭知识库检索
- `model`：不是模型名，而是 `GET /api/models` 返回的模型 UUID
- `appendUser`
  - 当前页面首个回答通常为 `true`
  - “追加回复”会传 `false`
- `contextSize`：放入 prompt 的历史消息条数

### 返回

```json
{
  "sessionId": "ses_xxx",
  "answer": "BM25 是一种相关性排序算法...",
  "answerHash": "sha256...",
  "citations": [
    {
      "docId": "doc_xxx",
      "filename": "公司制度.pdf",
      "chunkId": "chunk_xxx",
      "chunkIndex": 3,
      "sourceHint": "第 2 页",
      "snippet": "... BM25 ...",
      "kbId": "kb_xxx"
    }
  ]
}
```

**前端用途**

- 发送问题
- 渲染回答
- 渲染引用卡片
- 保存 `sessionId`
- 刷新会话列表时间

---

### 2) SSE 流式聊天（可选重写方案）

**接口**

`GET /api/chat/stream`

**常用查询参数**

- `sessionId`
- `message`（必填）
- `topK`
- `kbs`
- `useKb`
- `model`
- `appendUser`
- `contextSize`

**SSE 事件**

- `token`：增量文本
- `meta`：最终 `sessionId / answerHash / citations`

**说明**

- 当前页面实际主要走 `POST /api/chat`
- 如果你重写成现代前端，**很适合直接改成 SSE 流式体验**
- 如果用 `EventSource`，要依赖同源 Session Cookie；如果改用可自定义请求头的流式客户端，也可显式发送 `Authorization: Basic ...`

---

### 3) 追加回复

当前页面支持对同一条用户问题，切换模型再回答一次。

**仍然使用的接口**

- `POST /api/chat`

**区别**

- `message` 仍然是原问题
- `model` 改成新模型 UUID
- `appendUser=false`

**前端实现要点**

- 需要给每个“用户问题 turn”保留 `turnId`
- 需要保留原始 `promptText`
- 追加的回答要追加在同一个 turn 下，而不是新建用户消息

如果你想简化首版，可先不做“追加回复”，但建议文档中保留该能力。

## 6.4 模型模块

### 获取模型列表

**接口**

`GET /api/models`

**返回**

```json
{
  "models": [
    {
      "id": "f8b6...",
      "modelId": "openai/gpt-4o-mini",
      "displayName": "GPT-4o Mini",
      "providerName": "OpenRouter",
      "tags": ["chat"],
      "capabilities": ["text"]
    }
  ]
}
```

### 前端使用规则

当前前端会做两类筛选：

1. **回答模型**
   - 规则：`tags` 不包含 `embed` 且不包含 `rerank`
   - 用于聊天回答

2. **嵌入模型**
   - 规则：`tags` 包含 `embed`
   - 用于“我的知识库 -> 云端 API 嵌入”下拉框

### 重写建议

- 模型选择器建议保留：搜索、分组、标签、供应商名称
- 首版至少要支持：单选聊天模型
- 二期可以支持：多模型对照回答

## 6.5 知识库选择模块

### 获取当前用户可访问知识库

**接口**

`GET /api/kbs`

**返回**

```json
{
  "kbs": [
    {
      "id": "default",
      "name": "公司章程",
      "embeddingMode": "local",
      "embeddingModel": "",
      "embeddingBaseUrl": "",
      "createdAt": "2026-03-01T00:00:00Z",
      "updatedAt": "2026-03-16T00:00:00Z",
      "isDefault": true,
      "isPublic": true,
      "owned": true,
      "isSystem": true,
      "documentCount": 6
    }
  ]
}
```

### 字段含义

- `owned=true`：当前用户拥有 / 绑定了该知识库
- `isPublic=true`：管理员创建的“全体知识库”
- `isSystem=true`：系统默认知识库 `default`
- `documentCount`：该知识库默认检索条数配置

### 前端使用规则

- 默认应选中全部可访问知识库
- 关闭“启用知识库检索”时，聊天请求里应传空数组或直接禁用 KB
- 用户只能维护“自己拥有且不是 system/public 的私有知识库”

## 6.7 我的知识库模块

### 1) 创建私有知识库

**接口**

`POST /api/kbs`

**请求体**

```json
{
  "name": "个人项目库",
  "embeddingMode": "local",
  "embeddingModel": "",
  "embeddingBaseUrl": "",
  "documentCount": 6
}
```

### 说明

- `embeddingMode` 常见值：`local` / `api`
- 当 `embeddingMode=api` 时，可填写 `embeddingModel`
- 这里的 `embeddingModel` 建议使用后台已启用的嵌入模型 UUID

---

### 2) 修改私有知识库

**接口**

`PATCH /api/kbs/{kbId}`

**请求体**

### 说明

- 当前用户只能更新自己绑定的知识库
- 当前接口主要改：名称、默认检索条数

## 8. 前端必须处理的交互规则

## 8.1 登录态规则

- 所有 `fetch` 建议带 `credentials: 'same-origin'`
- 任意接口返回 `401` 时，弹登录框
- 登录成功后重新初始化页面

## 8.2 管理员跳转规则

- `GET /api/auth/me` 返回 `role=ADMIN` 时，当前页面会跳转 `/admin`
- 如果你以后打算允许管理员也用聊天页，需要明确是否保留这一跳转行为

## 8.3 知识库选择规则

- `useKb=false` 时，不应再带知识库检索
- `selectedKbIds=null` 表示“全部可访问知识库”
- `selectedKbIds=[]` 表示“完全不用知识库”
- 默认首次加载会选中全部知识库

## 8.4 模型选择规则

- 聊天回答使用“聊天模型”
- 嵌入模型只用于创建知识库时选择 embedding
- 不要把 embed/rerank 模型直接拿去聊天

## 8.5 消息渲染规则

- 一个 turn 至少包含 1 条用户消息
- 同一 turn 下可以挂多条 assistant 回答
- 每条 assistant 消息都可能有自己的模型名和引用列表

## 8.6 输入规则

- `Enter` 发送
- `Shift + Enter` 换行
- 发送中应禁用重复提交

---

## 9. 错误态与空态清单

重写时建议明确处理以下状态：

### 登录 / 权限

- `401`：登录失效，弹登录框
- `403`：账号待审批 / 无权限

### 聊天

- `知识库检索服务暂时不可用，请稍后再试。`
  - 含义：本次请求里的所有知识库检索都失败了
