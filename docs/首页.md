## 5. 当前页面状态模型

当前页面核心状态大致如下：

```js
{
  me,
  theme,
  menuOpen,
  registerAvatarDataUrl
}
```

说明：

- `me`：当前登录用户；未登录时为 `null`
- `theme`：`dark | light`
- `menuOpen`：移动端菜单是否展开
- `registerAvatarDataUrl`：注册弹窗里选择的头像 Data URL

## 6. 页面模块与所需接口

## 6.1 登录态识别与入口分流模块

### 1) 获取当前用户

**接口**

`GET /api/auth/me`

**返回常见字段**

```json
{
  "id": "uuid",
  "username": "alice",
  "displayName": "Alice",
  "avatarDataUrl": "data:image/png;base64,...",
  "role": "USER",
  "status": "ACTIVE"
}
```

**前端用途**

- 页面初始化时判断是否已登录
- 决定顶部主按钮文案
- 决定“进入系统”跳转到 `/chat` 还是 `/admin`
- 识别当前是否为管理员

**当前页面规则**

- 未登录或 `401` 时，首页保持游客态，不弹全局报错
- `role=ADMIN` 时主入口跳转 `/admin`
- 其他角色主入口跳转 `/chat`

---

### 2) 主入口跳转规则

**当前页面无新增接口**

页面使用本地规则 `routeByRole(me)`：

```js
role === 'ADMIN' ? '/admin' : '/chat'
```

**前端用途**

- 顶部“登录 / 进入系统 / 进入后台”按钮
- Hero 区“进入系统”按钮
- CTA 区“进入系统”按钮

---

## 6.2 登录模块

### 1) 登录

**接口**

`POST /api/auth/login`

**请求体**

```json
{
  "username": "alice",
  "password": "user123"
}
```

**返回**

返回内容与 `GET /api/auth/me` 基本一致，同时建立 Session Cookie。

**前端用途**

- 登录弹窗提交
- 登录成功后刷新按钮文案
- 按角色跳转到 `/chat` 或 `/admin`

**常见错误**

- `400`：账号或密码为空
- `401`：账号或密码错误
- `403`：账号待审批或已被禁用

---

### 2) 登录弹窗交互

**当前页面无新增接口**

**前端规则**

- 未登录时点击主入口，打开登录弹窗
- 点击遮罩或“取消”关闭弹窗
- 按 `Enter` 触发登录提交
- 按 `Esc` 关闭登录 / 注册弹窗

---

## 6.3 注册模块

### 1) 注册

**接口**

`POST /api/auth/register`

**请求体**

```json
{
  "username": "alice",
  "password": "user123",
  "displayName": "Alice",
  "avatarDataUrl": "data:image/png;base64,...",
  "inviteCode": "ACME2024"
}
```

**字段说明**

- `displayName`：可选昵称
- `avatarDataUrl`：可选头像 Data URL
- `inviteCode`：可选；填写后自动加入对应租户

**响应示例**

```json
{ "ok": true, "status": "PENDING" }
```

**前端用途**

- 注册弹窗提交
- 支持企业邀请码注册
- 注册成功后提示“等待管理员审批”

**常见错误**

- `400`：账号 / 密码不合法
- `409`：账号已存在

---

### 2) 注册头像预览

**当前页面无新增接口**

**前端规则**

- 支持 `png / jpg / jpeg / webp / gif`
- 文件读入后转为 Data URL 再提交
- 未上传头像时，使用账号 / 昵称生成文字头像
- 点击“移除头像”会清空文件与预览

---

## 6.4 退出登录与联系管理员模块

### 1) 退出登录

**接口**

`POST /api/auth/logout`

**返回**

```json
{ "ok": true }
```

**前端用途**

- 登录后，顶部次按钮从“注册”切换为“退出登录”
- 退出后恢复游客态按钮文案

---

### 2) 联系管理员

**当前页面无新增接口**

**前端规则**

- 未登录时点击“联系管理员”，当前实现会打开注册弹窗
- 已登录且 `role=ADMIN` 时，直接跳转 `/admin`
- 已登录普通用户时，弹出提示：联系系统管理员处理账号、权限或企业接入事宜

## 8. 前端必须处理的交互规则

## 8.1 登录态规则

- 所有接口请求建议带 `credentials: 'same-origin'`
- `GET /api/auth/me` 返回 `401` 时，首页保持游客态
- 登录成功后应立即按角色跳转，而不是停留在首页

## 8.2 按钮文案规则

- 未登录时：主按钮为“登录”，次按钮为“注册”
- 已登录普通用户时：主按钮为“进入系统”，次按钮为“退出登录”
- 已登录管理员时：主按钮为“进入后台”，次按钮仍为“退出登录”

## 8.3 路由跳转规则

- `ADMIN` → `/admin`
- 其他已登录角色 → `/chat`
- 首页本身只负责入口分流，不承载业务问答逻辑

## 8.4 主题与菜单规则

- 主题值从 `localStorage.kb_theme` 恢复
- 打开弹窗时应自动关闭移动端菜单
- 点击移动端导航项后应自动关闭菜单

## 8.5 表单规则

- 登录时必须同时填写账号和密码
- 注册时必须填写账号和密码
- 注册昵称、头像、邀请码均为可选项
- `Enter` 可直接提交登录 / 注册

## 8.6 弹窗规则

- 点击遮罩可关闭弹窗
- `Esc` 关闭登录弹窗、注册弹窗和移动菜单
- 注册弹窗打开时应刷新头像预览

---

## 9. 错误态与空态清单

### 登录 / 权限

- `401`：账号或密码错误
- `403`：账号待审批或已被禁用
- 首页初始化时 `GET /api/auth/me` 返回 `401`：按未登录处理

### 注册

- `409`：账号已存在，请更换账号名
- 输入不合法：提示检查账号、密码与邀请码
- 注册成功但 `status=PENDING`：提示等待管理员审批

### 头像处理

- 头像文件读取失败
- 头像格式不支持
- 头像过大导致 Data URL 提交失败

### 网络 / 服务

- 接口超时或服务不可用
- 非 JSON 错误返回导致的通用失败提示
