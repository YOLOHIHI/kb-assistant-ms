## 5. 当前页面状态模型

当前页面核心状态大致如下：

```js
{
  me,
  section,
  userFilter,
  kbId,
  kbs,
  embedModels,
  providers,
  providerId,
  models,
  modelsQueryProviderId,
  browser: {
    providerId,
    providerName,
    models,
    search,
    category,
    notice
  },
  theme,
  sidebarCollapsed,
  sidebarWidth,
  profileAvatarDataUrl,
  replyMsgId
}
```

说明：

- `section`：当前激活区块，取值为 `users | models | kb | tenants | messages`
- `userFilter`：用户状态筛选
- `kbId`：当前选中的后台知识库
- `embedModels`：当前可用于创建云端嵌入知识库的已启用模型
- `browser`：渠道模型浏览器弹窗状态
- `replyMsgId`：当前正在回复的留言 ID；代码里实际使用 `_replyMsgId`

## 6. 页面模块与所需接口

## 6.1 管理员登录与身份模块

### 1) 获取当前用户

**接口**

`GET /api/auth/me`

**返回常见字段**

```json
{
  "id": "uuid",
  "username": "admin",
  "displayName": "系统管理员",
  "avatarDataUrl": "data:image/png;base64,...",
  "role": "ADMIN",
  "status": "ACTIVE"
}
```

**前端用途**

- 页面初始化时校验是否已登录
- 校验 `role=ADMIN`
- 渲染顶部头像、身份标签和个人资料弹窗默认值

**当前页面规则**

- 未登录或登录失效时弹出管理员登录框
- 非管理员访问时按 `403` 处理，不展示后台内容

---

### 2) 管理员登录

**接口**

`POST /api/auth/login`

**请求体**

```json
{
  "username": "admin",
  "password": "admin123"
}
```

**前端用途**

- 管理员登录弹窗提交
- 登录成功后执行 `loadAfterLogin()`
- 自动加载当前激活区块的数据

**常见错误**

- `401`：账号或密码错误
- `403`：当前账号不是管理员，或账号待审批

---

### 3) 更新个人资料

**接口**

`PATCH /api/auth/profile`

**请求体**

```json
{
  "displayName": "系统管理员",
  "avatarDataUrl": "data:image/png;base64,..."
}
```

**说明**

- `displayName` 可为空字符串
- `avatarDataUrl` 可为空字符串，表示移除头像
- 支持 `png / jpg / jpeg / webp / gif` Data URL

**前端用途**

- 个人资料弹窗保存
- 更新顶部身份区头像与昵称

---

### 4) 退出登录

**接口**

`POST /api/auth/logout`

**返回**

```json
{ "ok": true }
```

**前端用途**

- 清理后台登录态
- 退出后跳回首页 `/`

---

## 6.2 导航、总览与布局模块

### 1) 后台区块切换

**当前页面无新增接口**

**前端规则**

- 侧栏点击后切换 `section`
- `section` 持久化在 `localStorage.kb_admin_section`
- 切换到不同区块时分别触发对应加载函数
  - `users` → `loadUsers()`
  - `models` → `loadModelSection()`
  - `kb` → `loadKbSection()`
  - `tenants` → `loadTenants()`
  - `messages` → `loadAdminMessages()`

---

### 2) 侧栏布局

**当前页面无新增接口**

**前端规则**

- 支持侧栏收起 / 展开
- 支持拖拽调整宽度，范围约为 `180 ~ 360px`
- `sidebarCollapsed` 与 `sidebarWidth` 持久化在 `localStorage`
- 移动端宽度下不启用拖拽拉伸

---

### 3) 主题切换

**当前页面无新增接口**

**前端规则**

- 支持 `light / dark`
- 与首页共用 `localStorage.kb_theme`
- 切换主题后需要重新渲染 Provider 卡片、模型表格与模型浏览器

---

## 6.3 用户审批模块

### 1) 查询用户列表

**接口**

`GET /api/admin/users`

**查询参数**

- `status`：可选，支持 `PENDING | ACTIVE | REJECTED | DISABLED`

**响应示例**

```json
{
  "users": [
    {
      "id": "uuid",
      "username": "alice",
      "displayName": "Alice",
      "role": "USER",
      "status": "PENDING",
      "createdAt": "2026-03-16T11:00:00Z"
    }
  ]
}
```

**前端用途**

- 用户表格渲染
- 顶部“待审批”徽标统计
- 用户状态筛选按钮切换

---

### 2) 审批通过用户

**接口**

`POST /api/admin/users/{id}/approve`

**前端用途**

- “通过”按钮
- 审批后刷新用户列表

**补充说明**

- 审批通过后，系统会确保用户至少拥有一个可访问知识库
- 系统知识库 `公司章程` 默认对所有用户可见

---

### 3) 驳回用户

**接口**

`POST /api/admin/users/{id}/reject`

**前端用途**

- “驳回”按钮
- 驳回前弹确认框

---

### 4) 禁用用户

**接口**

`POST /api/admin/users/{id}/disable`

**前端用途**

- 仅超管可见“禁用”按钮
- 仅 `ACTIVE` 状态的非管理员、且非当前登录管理员本人可禁用
- 禁用前弹确认框；禁用后刷新用户列表

**补充说明**

- 禁用不会删除用户数据
- 被禁用用户会进入“已禁用”筛选列表
- 被禁用用户再次登录会收到提示“您的账号已被禁用，请联系管理员”

---

### 5) 设为 / 撤销租户管理员

**接口**

- `POST /api/admin/users/{id}/tenant-admin`
- `DELETE /api/admin/users/{id}/tenant-admin`

**前端用途**

- 仅超管可见“设为租户管理员” / “撤销租户管理员”按钮
- 点击前弹出带账号名的确认框，避免误操作
- 成功后直接更新当前行的角色显示，不依赖整表刷新猜测结果

**补充说明**

- 只有已归属租户的非管理员用户可被设为租户管理员
- 接口会返回更新后的 `user` 行数据，前端据此只回写被操作账号

---

### 6) 恢复用户

**接口**

`POST /api/admin/users/{id}/restore`

**前端用途**

- 仅超管可见“恢复”按钮
- 仅 `DISABLED` 状态用户显示
- 恢复后刷新用户列表，并重新回到 `ACTIVE`

## 8. 前端必须处理的交互规则

## 8.1 登录态与权限规则

- 所有后台请求建议带 `credentials: 'same-origin'`
- 任意后台接口返回 `401` 时，应弹出管理员登录框
- `GET /api/auth/me` 返回非管理员时，应按 `403` 处理

## 8.2 区块切换规则

- 当前区块保存在 `localStorage.kb_admin_section`
- 切换区块时只加载对应数据，避免整页重复请求
- 用户筛选值、当前 Provider、当前知识库也都需要持久化

## 8.3 用户审批规则

- 待审批用户数应显示在侧栏徽标上
- 只有 `PENDING` 用户显示“通过 / 驳回”按钮
- 只有超管可以对 `ACTIVE` 用户执行“禁用”，对 `DISABLED` 用户执行“恢复”
- `ADMIN` 账号与当前登录管理员本人都不显示“禁用”按钮
- 审批后应立即刷新列表与徽标数量

## 8.4 模型治理规则

- 用户页 / 聊天页只会看到已启用模型
- 渠道模型弹窗支持分类筛选：推理 / 视觉 / 联网 / 免费 / 嵌入 / 重排 / 工具
- 远端模型拉取失败时，页面应回退展示本地已同步模型，并展示 notice

## 8.5 知识库规则

- `default` / `公司章程` 为系统默认知识库，不可删除
- 删除知识库时系统要求至少保留一个知识库
- `embeddingMode=api` 时必须选择嵌入模型
- 文档上传、删除、重建索引后要刷新统计与文档列表

## 8.6 租户规则

- 刷新邀请码后旧码立即失效
- 复制邀请码优先使用 Clipboard API，失败时退回 `textarea + execCommand`
- 创建租户时需要同时填写 `name` 与 `slug`

## 8.7 留言规则

- 回复内容不能为空
- 回复成功后状态自动变为 `REPLIED`
- 点击“关闭”前应有确认提示

## 8.8 个人资料与布局规则

- 主题与首页共用同一份 `kb_theme`
- 侧栏宽度范围要受限，避免布局异常
- 打开个人资料弹窗时应显示当前昵称 / 头像预览

## 9. 错误态与空态清单

### 登录 / 权限

- `401`：登录失效，弹管理员登录框
- `403`：当前账号不是管理员，或账号待审批

### 用户管理

- 暂无用户
- 用户列表加载失败
- 审批失败 / 驳回失败

### Provider / 模型

- 暂无 Provider
- Provider 添加失败
- Provider 删除失败
- 渠道模型远端加载失败，回退本地模型
- 无匹配模型
- 模型启用 / 停用失败

### 知识库 / 文档

- 暂无可用嵌入模型
- 后台知识库加载失败
- 系统默认知识库不可删除
- 上传失败
- 删除文档失败
- 重建索引失败

### 租户

- 暂无租户
- 创建失败
- 刷新邀请码失败
- 启用 / 停用失败

### 留言

- 暂无留言
- 留言加载失败
- 回复内容为空
- 回复失败
- 关闭失败
